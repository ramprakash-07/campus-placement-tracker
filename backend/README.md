# Campus Placement Tracker — Backend

FastAPI + SQLAlchemy + PostgreSQL backend for the Campus Placement Tracker.

---

## Prerequisites

- **Python 3.11+**
- **PostgreSQL 15+** running on `localhost:5432`
- A `.env` file in this directory (copy from `.env.example` and fill in your values)

## Setup

```bash
# Install dependencies
pip install -r requirements.txt

# Copy and configure environment variables
cp .env.example .env
# Edit .env with your DATABASE_URL, SECRET_KEY, etc.
```

## Database Migrations (Alembic)

This project uses [Alembic](https://alembic.sqlalchemy.org/) for database schema migrations. All migration scripts live in `alembic/versions/`.

### Configuration

- **`alembic.ini`** — Alembic settings (logging, script location). The database URL is **not** hardcoded here; it is set programmatically from `DATABASE_URL` in your `.env`.
- **`alembic/env.py`** — Imports the SQLAlchemy `Base` and all ORM models so that `--autogenerate` can detect schema changes.

### Common Commands

| Command | Description |
|---|---|
| `alembic upgrade head` | Apply all pending migrations |
| `alembic downgrade -1` | Roll back the last migration |
| `alembic downgrade base` | Roll back **all** migrations |
| `alembic revision --autogenerate -m "description"` | Generate a new migration from model changes |
| `alembic revision -m "description"` | Create an empty migration script |
| `alembic current` | Show the current migration revision |
| `alembic history` | List all migration revisions |

### Workflow

1. **Modify a model** in `models/` (e.g. add a column, change a type).
2. **Generate a migration**:
   ```bash
   alembic revision --autogenerate -m "add avatar_url to users"
   ```
3. **Review** the generated file in `alembic/versions/` — always verify autogenerated code.
4. **Apply** the migration:
   ```bash
   alembic upgrade head
   ```

### First-time Setup

If the database has no tables yet, apply the initial migration:

```bash
alembic upgrade head
```

## Running the Server

```bash
uvicorn main:app --reload
```

The API will be available at `http://localhost:8000`. Interactive docs at `/docs`.
